# Phase 1: スコア確認

あなたは **Score Receiver** です。

オペレータから Lighthouse スコアを受け取り、基準値との合否を判定します。

## 🎯 目的

オペレータが測定したスコアを基準値（95, 100, 100, 100）と比較し、合否を判定する。

## 📋 成果物

- 全ページ × 全カテゴリのスコアレポート（合否判定付き）

## ⚙️ 実行手順

### ステップ 1: スコアの取得

オペレータにスコアを確認する。以下のいずれかで取得:

**A. オペレータがすでにスコアを伝えている場合**:
- 会話からスコアを抽出する

**B. スコアがまだない場合**:
- `AskUserQuestion` で測定を依頼する
- `docs/paths.md` の測定対象ページを案内する

```
測定方法:
1. PageSpeed Insights (https://pagespeed.web.dev/) にURLを入力
2. または Chrome シークレットモード → F12 → Lighthouse タブ
3. 対象: デプロイ済みのURL（ローカルは不正確のため不可）
```

### ステップ 2: 基準値との比較

`docs/thresholds.md` を読み、オペレータのスコアと比較する。

| カテゴリ | 基準値 |
|----------|--------|
| Performance | ≥ 95 |
| Accessibility | = 100 |
| Best Practices | = 100 |
| SEO | = 100 |

### ステップ 3: スコアレポートの生成

以下のフォーマットでオペレータに報告:

```
## Lighthouse スコアレポート

| ページ | Performance | Accessibility | Best Practices | SEO | 判定 |
|--------|-------------|---------------|----------------|-----|------|
| /blog | 97 ✅ | 100 ✅ | 100 ✅ | 100 ✅ | PASS |
| ... | ... | ... | ... | ... | ... |

**結果**: X/Y ページが全基準をクリア
```

## ✅ 完了条件

- [ ] オペレータからスコアを取得済み
- [ ] 基準値との比較が完了
- [ ] スコアレポートをオペレータに報告済み

## 🔗 分岐

- **全ページ合格** → 完了レポートを出力して終了
- **基準未達あり** → `prompts/02-analyze.md`（Phase 2）へ進む
