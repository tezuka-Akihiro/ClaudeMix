# 参照ファイル一覧

ステップ1で読み込む4つの参照ファイルの一覧と、それぞれの役割。

## 必須参照ファイル（4つ）

### 1. ブランド・マニフェスト

**ファイルパス**: `docs/ブランド/ブランド・マニフェスト.md`

**役割**: ClaudeMixプロジェクトの根幹思想、ターゲットペルソナ、ブランドペルソナを定義

**確認項目**:

- **根幹思想**:
  - 第一目的: 「優しい人が、優しいまま生きていける世界まで、バトンをつなぐ」
  - 行動哲学: 「最大の努力で、最小の結果（＝死なないこと）を生む」

- **ターゲットペルソナ**:
  - AI活用で自立を目指す非技術者
  - 中堅エンジニア（5年目前後）

- **ブランドペルソナ**:
  - 頼れる建築家（信頼性）
  - 現実的な先生（実践性）
  - 用心棒（守護）

**記事への反映**:
- ターゲットペルソナを意識した書き方
- 「優しい人が、優しいまま生きていける」という価値観を記事に反映
- 「最大の努力で、最小の結果」という行動哲学を記事に反映

## 読み込みの順序

ステップ1では、以下の順序で読み込んでください：

1. **ブランド・マニフェスト** → 根幹思想の理解
2. **プロフィール** → 筆者の経験・哲学の理解
3. **プロダクト設計書** → コンセプト・技術スタックの理解
4. **コンテキスト** → ストーリーテリング手法の理解

この順序により、プロジェクトの全体像を段階的に理解できます。

## 参照ファイルの更新

参照ファイルが更新された場合、以下の点に注意してください：

- **ブランド・マニフェスト**: 根幹思想、ターゲットペルソナの変更は記事に大きな影響
- **プロフィール**: 経験・哲学の更新は記事のトーンに影響
- **プロダクト設計書**: 技術スタックの変更は記事の内容に影響
- **コンテキスト**: ストーリーテリング手法の変更は導入部分に影響

参照ファイルが更新された場合は、最新のファイルを読み込んでください。

## よくある質問

### Q1: すべての参照ファイルを毎回読み込む必要がありますか？

**A**: はい、ステップ1では4つすべての参照ファイルを読み込んでください。プロジェクトのコンセプトを理解するために必要です。

### Q2: 参照ファイルが見つからない場合はどうすれば良いですか？

**A**: ファイルパスを確認してください。パスが正しい場合は、オペレーターに確認してください。

### Q3: 参照ファイルの内容が矛盾している場合はどうすれば良いですか？

**A**: オペレーターに確認してください。基本的には、ブランド・マニフェストが最上位の方針となります。
