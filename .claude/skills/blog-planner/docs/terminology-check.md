# プロダクト固有用語チェック

読者が予備知識なしで理解できる表現を使用するためのチェックリスト。

## 基本方針

**原則**: プロダクト固有の用語を避け、一般的な用語に置き換える。

**理由**:
- 読者は「ClaudeMix」というプロダクト名を知らない
- プロダクト固有の用語は読者に予備知識を要求し、離脱を招く
- 一般的な用語に置き換えることで、読者の理解を容易にする

## チェックプロセス

### ステップ1: NG用語の検出

記事の中から、以下のNG用語を検索してください：

- ClaudeMix
- このブログ / 当ブログ（文脈に応じて）
- Projects（Claudeの）
- Skills（Claudeの）
- .claude（補足なし）
- claude-config.json（補足なし）
- Slash Commands
- OperatorBot
- 3大層アーキテクチャ（補足なし）
- lib層 / data-io層（補足なし）

### ステップ2: 代替案の適用

NG用語が検出された場合、上記の代替案を適用してください。

### ステップ3: 補足説明の追加

一般的な用語に置き換えられない場合は、補足説明を追加してください。

**例**:

**NG**:
「.claude ディレクトリに設定ファイルを配置します。」

**OK**:
「Claude Codeの設定ディレクトリ（.claude/）に設定ファイルを配置します。」

## 例外

以下の場合は、プロダクト固有用語を使用しても問題ありません：

1. **固有名詞として明示的に紹介する場合**
   - 「ClaudeMixというブログを運営しています」

2. **コードスニペット内**
   - ファイルパス、ディレクトリ名等

3. **技術的な正確性のために必要な場合**
   - Claude Code Projectsの機能を紹介する際の「Projects」

**例**:

**OK（固有名詞として紹介）**:
「筆者はClaudeMixというRemix × Cloudflare Edgeのブログを運営しています。」

**OK（コードスニペット内）**:
```json
// .claude/claude-config.json
{
  "name": "ClaudeMix"
}
```

**OK（技術的な正確性）**:
「Claude Code Projectsは、プロジェクト固有の設定を.claude/claude-config.jsonファイルで管理します。」

## よくある質問

### Q1: すべてのプロダクト固有用語を避ける必要がありますか？

**A**: はい、読者が予備知識なしで理解できるよう、プロダクト固有用語は避けてください。ただし、固有名詞として明示的に紹介する場合や技術的な正確性のために必要な場合は例外です。

### Q2: 「このブログ」という表現は使えませんか？

**A**: 文脈に応じて使用できます。ただし、「ClaudeMix」というプロダクト名と混同されないよう、「筆者が運営するブログ」「Remix × Cloudflare Edge環境」等の表現を推奨します。

### Q3: コードスニペット内のプロダクト固有用語もチェックが必要ですか？

**A**: いいえ、コードスニペット内のファイルパス、ディレクトリ名等は変更不要です。
