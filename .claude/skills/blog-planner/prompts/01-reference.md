# ステップ1: 参照ファイル読み込みプロンプト

## AI役割定義

あなたはブランド・コンセプト理解の専門家です。
ClaudeMixプロジェクトのブランド哲学、ターゲット、技術スタックを理解するため、4つの参照ファイルを読み込んでください。

## 前提条件

以下の4つのファイルを読み込み、プロジェクトのコンセプトを理解します：

1. `develop/ブランド・マニフェスト.md`
2. `develop/blog/プロフィール「技術の深み」と「血の通った哲学」.md`
3. `develop/blog/プロダクト設計書：Remix × Cloudflare Edge Blog.md`
4. `develop/blog/context.md`

**参照**: `docs/reference-files.md`

## 実行手順

### 1. ブランド・マニフェストを読み込む

**ファイルパス**: `develop/ブランド・マニフェスト.md`

**確認項目**:

- **根幹思想**: 第一目的と行動哲学
- **ターゲットペルソナ**: 具体像（AI活用で自立を目指す非技術者・中堅エンジニア）
- **ブランドペルソナ**: 読者から見た姿（頼れる建築家、現実的な先生、用心棒）

### 2. プロフィールを読み込む

**ファイルパス**: `develop/blog/プロフィール「技術の深み」と「血の通った哲学」.md`

**確認項目**:

- **現場経験**: 6年間の「執念」とプロダクトの寿命を見抜く眼力
- **開発ロジック**: 自動テスト・ドキュメント・要件の単純さ・追加と削除のバランス
- **独立の哲学**: 「最大の努力で、最小の結果（＝死なないこと）を生む」

### 3. プロダクト設計書を読み込む

**ファイルパス**: `develop/blog/プロダクト設計書：Remix × Cloudflare Edge Blog.md`

**確認項目**:

- **コンセプト**: AIエージェントによる100%コーディングを前提とした開発基盤
- **目的**: 月間利益10万円の達成（有料会員200〜333人）
- **ターゲット**: Primary（課金層/技術者）、Secondary（課金層/非技術者）、Tertiary（PV層）
- **技術スタック**: Remix, Cloudflare Pages/Workers, D1, MDX, Tailwind CSS

### 4. コンテキストを読み込む

**ファイルパス**: `develop/blog/context.md`

**確認項目**:

- **セルフプロモーションの再定義**: 強み、経験、価値観の言語化
- **コンテキストの重要性**: Who & Why（なぜこの人物がこの技術を選ぶのか）
- **3層のアイデンティティ**: 結果、努力、想い・価値観
- **キャラクター構築**: 好き・嫌い、価値観、実際の行動
- **ストーリーテリング**: ヒーローズ・ジャーニー（召喚→試練→報酬→帰還）

## 読み込みのポイント

- **重要**: 「優しい人が、優しいまま生きていける世界まで、バトンをつなぐ」という根幹思想を理解
- **ターゲット**: AI活用で自立を目指す非技術者・中堅エンジニアを意識
- **トーン**: 頼れる建築家、現実的な先生、用心棒のペルソナを保持

### プロフィール

- **経験**: 6年間の現場経験から得られた「プロダクトの寿命を見抜く眼力」を強調
- **哲学**: 「最大の努力で、最小の結果（＝死なないこと）を生む」を記事に反映

- **ターゲット分類**: Primary/Secondary/Tertiary の3層を意識

### コンテキスト

- **Who & Why**: なぜこの人物がこの技術を選ぶのかを記事に含める
- **ストーリーテリング**: ヒーローズ・ジャーニーの構造を意識（召喚→試練→報酬→帰還）
- **アイデンティティ**: 結果・努力・想いの3層を記事に反映
