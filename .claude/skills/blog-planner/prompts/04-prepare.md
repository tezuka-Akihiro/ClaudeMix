## 実行手順

### 1. 恐怖型BAB法の要素を抽出

**詳細**: `docs/bab-structure.md`

恐怖型BAB法は、読者の「恐怖」を刺激し、解決策への期待を高めるコンテンツマーケティング手法です。

#### Before（課題）

**質問**:
- 読者が直面している課題は何か？
- その課題を放置すると、どのような恐怖が待っているか？
- なぜその課題は解決が難しいのか？

**ポイント**:
- **恐怖の明確化**: 読者が共感できる具体的な恐怖
- **課題の深刻さ**: 放置すると致命的な問題になることを示す
- **解決の困難さ**: 簡単には解決できない理由を示す

#### After（解決後の世界）

**質問**:
- 課題を解決すると、どのような世界が待っているか？
- 読者は何を得られるのか？
- どのような恐怖から解放されるのか？

**ポイント**:
- **理想の世界**: 課題が解決された後の具体的な状態
- **得られるもの**: 読者が手に入れる価値
- **恐怖からの解放**: Before の恐怖が消えることを示す

#### Bridge（解決策）

**質問**:
- Before から After に到達するための解決策は何か？
- この記事で紹介する手法は何か？
- なぜその解決策が効果的なのか？

**ポイント**:
- **具体的な解決策**: この記事で紹介する手法
- **効果の根拠**: なぜその解決策が効果的なのか
- **実践可能性**: 読者が明日から試せる

### 2. プロダクト固有用語をチェック

**詳細**: `docs/terminology-check.md`

読者が予備知識なしで理解できる表現を使用してください。

**チェック項目**:

| NG | OK | 理由 |
| :--- | :--- | :--- |
| ClaudeMix | Remix × Cloudflare Edgeのブログ | プロダクト名を避ける |
| このブログ | 筆者が運営するブログ | プロダクト名を避ける |
| .claude ディレクトリ | Claude Codeの設定ディレクトリ | 予備知識を要求しない |
| Projects ファイル | プロジェクト設定ファイル | 予備知識を要求しない |
| Skills | カスタムツール | 予備知識を要求しない |

**基本方針**:
- **プロダクト固有の用語を避ける**: ClaudeMix → Remix × Cloudflare Edgeのブログ
- **一般的な用語に置き換える**: Skills → カスタムツール
- **補足説明を追加する**: Projects → プロジェクト設定ファイル（.claude/claude-config.json）

### 3. 導入部分の構成を準備

Before/After/Bridgeを導入部分に組み込む構成を準備してください。

**構成例**:

```markdown

{その課題を放置すると待っている恐怖を1段落で記述}

## After（解決後の世界）

{課題を解決した後の理想の世界を1-2段落で記述}

## Bridge（解決策）

{この記事で紹介する解決策を1-2段落で記述}
```

## 恐怖型BAB法のポイント

### Before（課題）の書き方

**恐怖の明確化**:
- 「このまま放置すると、〜という致命的な問題が発生する」
- 「〜が原因で、プロダクトの寿命が縮まる」

**課題の深刻さ**:
- 具体的な数値や事例で示す
- 読者が「これは自分のことだ」と感じる

**解決の困難さ**:
- 「公式ドキュメントには書いていない」
- 「既存の方法では解決できない」

### After（解決後の世界）の書き方

**理想の世界**:
- Before の恐怖が消えた後の具体的な状態
- 測定可能な結果（Lighthouseスコア等）

**得られるもの**:
- 「〜の時間を節約できる」
- 「〜の品質が向上する」

### Bridge（解決策）の書き方

**具体的な解決策**:
- この記事で紹介する手法を明示
- 「〜を使うことで、〜を実現する」

**効果の根拠**:
- 「本番環境で検証済み」
- 「6年間の現場経験から得られた知見」

**実践可能性**:
- 「明日から試せる」
- 「再現可能なコードスニペット」
