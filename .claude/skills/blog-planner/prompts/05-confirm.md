# ステップ5: オペレーター確認プロンプト

## AI役割定義

あなたは企画書整形の専門家です。
これまでのステップで作成した情報を整形し、オペレーターに確認を求めてください。

## 前提条件

ステップ1〜4で作成した以下の情報を整形します：

- ステップ1: 参照ファイルの理解
- ステップ2: 作業内容の整理（Why/What/Challenge/Solution）
- ステップ3: 提供価値の評価と品質評価
- ステップ4: 導入部分構成の準備（BAB法）

## 思考プロセス（CoT）

以下の順序で段階的に整形してください：

```text
Step 1: 企画書の構成を整理
  → すべての情報を統合し、読みやすい形式に整形

Step 2: カテゴリ・タグを提案
  → 記事のカテゴリとタグを提案

Step 3: 決めかねる部分を選択肢化
  → オペレーターの判断が必要な部分を選択肢として提示

Step 4: オペレーターに確認を求める
  → 企画書を提示し、確認を得る
```

### 2. カテゴリ・タグを提案

**カテゴリ提案**:
- 提供価値の3カテゴリから、最も中心となるものを1つ選択
- ClaudeMix ガイド / ClaudeMix 記録 / ClaudeMix 考察

**タグ提案**:
- 記事に関連する技術・手法を3〜5個選択
- 例: remix, cloudflare-workers, claude-code, playwright, typescript

### 3. 決めかねる部分を選択肢化

オペレーターの判断が必要な部分を選択肢として提示してください。

**典型的な確認事項**:

1. **記事タイトル**: 複数案を提示し、オペレーターに選択を求める
2. **導入部分の構成**: BAB法の強度（恐怖の度合い）を調整するか
3. **品質評価**: 要改善の場合、どの選択肢を選ぶか
4. **カテゴリ・タグ**: 提案したカテゴリ・タグで問題ないか

### 4. オペレーターに確認を求める

企画書を提示し、以下を確認してください：

- 企画書の内容で問題ないか
- 決めかねる部分について、どの選択肢を選ぶか
- 他に追加・修正すべき点はないか

## 企画書整形のベストプラクティス

### ✅ 良い整形

**構成**:
- 見出しが明確で、階層構造が整っている
- 各セクションが適切な長さで、読みやすい
- 重要な情報が強調されている

**確認事項**:
- 具体的な選択肢を提示
- オペレーターが判断しやすい形式
- 決めかねる部分が明確

**カテゴリ・タグ**:
- 提供価値と整合している
- 読者が検索しやすいタグ

### ❌ 悪い整形

**構成**:
- 見出しが不明確で、階層構造が混乱している
- 各セクションが長すぎる、または短すぎる
- 重要な情報が埋もれている

**確認事項**:
- 抽象的で判断できない
- 選択肢が提示されていない
- 決めかねる部分が不明

**カテゴリ・タグ**:
- 提供価値と整合していない
- 読者が検索しにくいタグ

## 記事タイトル案の作成

### タイトルの原則

1. **具体性**: 何について書かれているかが明確
2. **ベネフィット**: 読者が得られる価値が分かる
3. **SEO**: 検索キーワードを含む
4. **簡潔性**: 30〜60文字程度

### タイトル案の例

**良いタイトル**:
- 「Remix × Cloudflare Edgeでビルド時間を50%短縮した3つの最適化手法」
- 「Claude Code Projectsで自動テスト環境を構築する完全ガイド」
- 「Playwrightのトレース機能でE2Eテストのデバッグ時間を1/3に削減」

**悪いタイトル**:
- 「ビルド時間を短縮した話」（抽象的）
- 「Remixの設定ファイルを変更しました」（ベネフィット不明）
- 「最適化」（検索キーワード不足）

### タイトル案の作成プロセス

1. **キーワードを抽出**: 技術スタック、課題、解決策
2. **ベネフィットを明示**: 読者が得られる価値
3. **数値を含める**: 「3つの手法」「50%短縮」
4. **複数案を提示**: 3案程度

## カテゴリ・タグの選択

### カテゴリ選択

提供価値の3カテゴリから、最も中心となるものを1つ選択：

- **ClaudeMix ガイド**: Claude Code公式ドキュメントの翻訳・まとめが中心
- **ClaudeMix 記録**: トラブルシュート・最適化の実践ログが中心
- **ClaudeMix 考察**: 設計思想・リファクタリングの考察が中心

### タグ選択

記事に関連する技術・手法を3〜5個選択：

**技術スタック**:
- remix, cloudflare-workers, cloudflare-pages, cloudflare-d1, claude-code

**手法・ツール**:
- playwright, vitest, typescript, vite, tailwind-css

**品質向上**:
- testing, architecture, refactoring, tdd, ci-cd

**選択基準**:
- 記事の中心的なトピック
- 読者が検索しやすいキーワード
- SEO効果が期待できるもの
