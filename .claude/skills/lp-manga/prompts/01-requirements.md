# Phase 1: 要件定義プロンプト

## AI役割定義

あなたはビジネスアナリスト兼UXリサーチャーです。
LP漫画の制作に向けて、ターゲットユーザーのペルソナとゴール（KPI）を明確化してください。

## 思考プロセス（CoT）

以下の順序で段階的に考えてください：

```text
Step 1: ターゲットの「最も深刻な悩み」を特定する
  → 何に困っているか？ どんな感情を抱いているか？

Step 2: その悩みが解決された「理想の状態」を描写する
  → どうなれば幸せか？ 何が変わるか？

Step 3: 1と2のギャップを埋める「心理的ハードル」を洗い出す
  → なぜ今まで解決できなかったか？ 何が障壁か？

Step 4: ハードルを越えさせる「訴求ポイント」を導出する
  → 何があれば行動に移せるか？
```

## 実行手順

### Step 1: ターゲット・ペルソナ定義

以下のテーブルを埋めてください。**具体的な固有名詞**を使用すること。

| 項目 | 内容 |
| :--- | :--- |
| **ターゲット属性** | 年齢、性別、職種、居住地など |
| **現状の悩み（負債）** | 具体的な困りごと、痛み |
| **心理的ハードル** | 行動を阻む不安や障壁 |
| **理想の状態（解決後）** | 製品/サービス導入後のありたい姿 |

**良い例**:

| 項目 | 内容 |
| :--- | :--- |
| ターゲット属性 | 32歳男性、SaaS企業のインサイドセールス、渋谷区在住、年収500万円 |
| 現状の悩み | リードへの架電数がKPIだが、リスト作成に1日2時間かかり架電できない |
| 心理的ハードル | 新しいツールの導入は上長の承認が面倒、使いこなせるか不安 |
| 理想の状態 | リスト作成が自動化され、架電に集中でき、成約率が上がって評価される |

**悪い例**:

| 項目 | 内容 | 問題点 |
| :--- | :--- | :--- |
| ターゲット属性 | 30代男性、営業職 | 抽象的すぎる |
| 現状の悩み | 仕事が大変 | 具体性がない |
| 心理的ハードル | 特になし | 深掘りが足りない |
| 理想の状態 | 楽になりたい | 曖昧 |

### Step 2: ゴール定義（KPI）

以下を明確に定義してください。

- **メインゴール（CV）**: 読後に取ってほしい1つのアクション
  - 例: 資料請求、無料トライアル申込、ホワイトペーパーDL、LINE登録

- **読後感**: 漫画を読み終わった時の感情
  - 例: 「これなら自分の課題を解決できるかも」という期待感

- **訴求ポイント**: CVに至らせる決め手（3つまで）
  - 例: 導入コストの低さ、即効性、サポートの手厚さ

## 完了条件チェックリスト

- [ ] ターゲット属性に「年齢」「職種」「具体的な悩み」が含まれている
- [ ] CVが1つに絞られている
- [ ] 読後感が「〜という気持ち」の形で言語化されている
- [ ] 訴求ポイントが3つ以内に絞られている
