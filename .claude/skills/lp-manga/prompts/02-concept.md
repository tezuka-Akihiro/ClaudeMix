# Phase 2: 基本設計プロンプト

## AI役割定義

あなたはストーリーテラー兼コピーライターです。
要件定義書を基に、LP漫画のストーリープロットとトーン＆マナーを設計してください。

## 前提条件

Phase 1で作成した以下が存在すること：

- ターゲット・ペルソナ定義書
- ゴール定義書

## 思考プロセス（CoT）

以下の順序で段階的に考えてください：

```text
Step 1: ターゲットの「現状」を映像として想像する
  → どんな場所で、どんな表情で、何をしているか？

Step 2: 「理想の状態」を映像として想像する
  → どんな場所で、どんな表情で、何をしているか？

Step 3: 現状→理想への「変化のきっかけ」を特定する
  → 何がトリガーで変わるか？ 誰が助けるか？

Step 4: PASONA法則に沿って感情の流れを設計する
  → 各段階で主人公はどんな感情か？
```

## 実行手順

### Step 1: ストーリー・プロット（PASONA法則）

全体の流れを「状態の変化」で記述します。セリフではなく、シーンと感情の変化を書いてください。

| 構成 | 段階 | 内容（シーン・状態の変化） |
| :--- | :--- | :--- |
| **P** | Problem（問題提起） | 主人公が困っている状況を描写 |
| **A** | Affinity（親近感） | 読者が「あるある」と共感する心理描写 |
| **S** | Solution（解決策） | 解決策が提示される瞬間（まだ体験前） |
| **O** | Offer（提案） | 解決策を使った後の世界（体験後） |
| **N** | Narrowing down（絞込） | 今行動すべき理由、限定性 |
| **A** | Action（行動） | CVへの誘導、ハッピーエンド |

**良い例**:

| 構成 | 内容 |
| :--- | :--- |
| **P** | 深夜23時、残業中の主人公。リスト作成が終わらず焦りの表情。同僚はすでに帰宅し、フロアに一人だけ。 |
| **A** | 「またこの作業か...」という徒労感。画面に映る無数のExcelセル。頭を抱える主人公。 |
| **S** | 先輩が登場。「まだそんなやり方してるの？」とスマホ画面を見せる。新ツールのロゴが光る。 |
| **O** | ワンクリックでリストが完成。驚く主人公。「え、これだけ？」という表情。時計は19時を指している。 |
| **N** | 「今なら14日間無料」のバナー。導入企業のロゴが並ぶ。「業界シェアNo.1」の文字。 |
| **A** | 笑顔で退社する主人公。「もっと早く使えばよかった！」→ CVボタンが画面下に。 |

**悪い例**:

| 構成 | 内容 | 問題点 |
| :--- | :--- | :--- |
| **P** | 主人公が困っている | 具体的な状況がない |
| **S** | ツールを使ってリストが完成した | SとOが混同されている（これはOの内容） |
| **O** | 「これで楽になりますね」 | 体験後の変化が描写されていない |

**注意**:

- SとOを混同しない: Sは「提示の瞬間」、Oは「使った後の世界」
- 各段階で主人公の感情が変化すること

### Step 2: トーン＆マナー定義

システムの「アーキテクチャ選定」に相当します。

- **画風スタイル**:
  - 日本のビジネス漫画風
  - アメコミ風
  - フラットデザイン
  - 水彩画風
  - その他（具体的に）

- **色彩設計**:
  - ブランドカラーがあれば基調に
  - 問題シーンと解決シーンで色調を変える

- **雰囲気**:
  - 誠実・信頼感重視
  - コミカル・勢い重視
  - 共感・エモーショナル重視

### Step 3: デリバリー仕様書（媒体定義）

最終的な出力形式を定義します。

- **表示形式**:
  - スマホ特化の縦読みスクロール
  - PC兼用の横読みスライド
  - Instagram用カルーセル
  - その他

- **画像サイズ**:
  - 推奨: 横1080px × 縦1350px（アスペクト比 4:5）
  - スマホ縦読み: 横1080px × 縦1920px（9:16）

- **ファイル形式**:
  - WebP（推奨、軽量）
  - PNG（透過が必要な場合）
  - JPG（写真素材が多い場合）
