# 失敗パターン集

このドキュメントは、LP漫画制作で発生した失敗パターンとその対処法を蓄積します。
運用を通じて更新し、ナレッジベースとして活用してください。

---

## AI画像生成の失敗パターン

### 指・手の問題

| 症状 | 発生頻度 | 対処法 |
| :--- | :--- | :--- |
| 指が6本になる | 高 | 「手は描かない」or「手は画面外」の構図に変更 |
| 指が3本になる | 中 | 同上 |
| 関節が逆に曲がる | 中 | バストアップ（胸から上）に変更 |
| 手が溶けている | 中 | クローズアップ（顔のみ）に変更 |

**プロンプト例**:
```text
NG: 全身が見える構図で、手を広げている
OK: バストアップの構図、手は画面外に配置
```

### キャラクター一貫性の問題

| 症状 | 発生頻度 | 対処法 |
| :--- | :--- | :--- |
| 顔が別人になる | 高 | 三面図を毎回添付、特徴タグを冒頭に配置 |
| 髪型が変わる | 高 | 髪型の詳細を特徴タグに追加 |
| 服装が変わる | 中 | 服装の詳細を特徴タグに追加 |
| 眼鏡が消える | 高 | 「必ず眼鏡をかけている」と明記 |

**プロンプト例**:
```text
NG: 黒髪の男性
OK: 黒髪のショートカット（前髪は眉にかかる程度、サイドは耳が半分見える長さ）、
    黒縁の四角い眼鏡を必ずかけている、30代前半の日本人男性
```

### 背景の問題

| 症状 | 発生頻度 | 対処法 |
| :--- | :--- | :--- |
| 意図しないロゴが出現 | 中 | 「背景にロゴや文字を入れない」と明記 |
| 場所が変わる | 中 | 背景の詳細を具体的に記述 |
| 時間帯がずれる | 低 | 「深夜」「日中」など時間帯を明記 |

## ストーリー設計の失敗パターン

### PASONA法則の混同

| 症状 | 対処法 |
| :--- | :--- |
| SとOが混同される | Sは「提示の瞬間」、Oは「使った後の世界」と明確に区別 |
| Aが弱い | 読者が「自分のことだ」と思える具体的なあるあるを追加 |
| Nが唐突 | 前のコマで「でも高いんでしょ？」などの心理的障壁を描写 |

**具体例**:
```text
NG:
S: ツールを使ってリストが完成した
O: 「これで楽になりますね」

OK:
S: 先輩が「このツール知ってる？」とスマホを見せる（まだ使ってない）
O: ワンクリックでリストが完成し、主人公が驚いている（使った後）
```

## 媒体・技術の失敗パターン

### ファイルサイズの問題

| 症状 | 対処法 |
| :--- | :--- |
| 画像が重くてLPが遅い | PNG→WebPに変換、TinyPNGで圧縮 |
| 画質が劣化しすぎ | 圧縮率を調整（品質80%程度を推奨） |

### 表示の問題

| 症状 | 対処法 |
| :--- | :--- |
| スマホで文字が読めない | 吹き出し内のフォントを14px以上に |
| 縦長画面で見切れる | アスペクト比を確認（4:5 or 9:16推奨） |
