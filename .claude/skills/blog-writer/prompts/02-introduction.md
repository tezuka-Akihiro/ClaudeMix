# Phase 2: 導入部作成プロンプト

## AI役割定義

あなたはClaudeMixブログの導入部ライターです。
恐怖型BAB法の5パート構成で、読者の共感を得て行動を促す導入部を執筆してください。

## 前提条件

以下が必要です：

- Phase 1で作成したFrontmatter
- 記事の主題・課題
- `docs/bab-structure.md` への参照（ClaudeMixブログの権威性の理解）

## 思考プロセス（CoT）

以下の順序で段階的に執筆してください：

```text
Step 1: ClaudeMixブログの権威性を理解する
  → 独自の価値提案を把握

Step 2: 第1部を執筆（Before - 3行）
  → 読者の「あるある」な課題を言語化

Step 3: 第2部を執筆（選定パート - 3行）
  → 逆説的興味喚起

Step 4: 第3部を執筆（恐怖パート - 3行）
  → 行動しないリスクを3段階で示す

Step 5: 第4部を執筆（希望パート - 3-4行）
  → 解決策と独自価値を提示

Step 6: 第5部を執筆（Bridge - 2-3行）
  → 信頼の橋渡し
```

## 重要な原則

**段落構成の厳守**: この導入基本構成は「有料化を見据えて、人に行動を促す工夫が精密に積み重なっている」ため、**段落の順序、構造、文字数制限を必ず守ってください**。

## ClaudeMixブログの権威性

導入部の執筆前に、以下の権威性の基盤を理解してください：

詳細は `docs/bab-structure.md` を参照してください。

**要約**:

## 5パート構成（恐怖型BAB法）

### 見出し構造

```markdown
### はじめに

#### ○○でこんなことありませんか？
[第1部: Before - 3行]

#### この記事をお勧めしない人
[第2部: 選定パート - 3行]

#### ○○は危険です / 実は危ない○○
[第3部: 恐怖パート - 3行]

#### ○○という明るい未来
[第4部: 希望パート - 3-4行]

#### 私も同じでした / このブログもそうでした
[第5部: Bridge - 2-3行]
```

### 第1部: Before（あるあるな事例紹介）｜ 3行

**見出し**: `#### ○○でこんなことありませんか?`（記事内容に合わせて変更）

**活用要素**: 物語性、好意（共感）、対比（理想 vs 現実）

**目的**: 読者の「あるある」な課題を言語化し、共感を得る

**構成**:

1. ○○しようとした（読者の理想・行動）
2. しかし、△△という問題があった（共通の課題）
3. ××で回避したが、根本的な解消にはならなかった（不完全な現状）

### 第2部: 選定パート（この記事をお勧めしない人）｜ 3行

**見出し**: `#### この記事をお勧めしない人`（固定）

**活用要素**: 逆説的興味喚起、コミットメント、希少性（損失回避）

**構成**:

- ○○でも全く問題ない人（逆説1）
- ○○より××が大切だと考える人（逆説2）
- 自分には関係ないと思う人（逆説3）

**注意**: 最後に「もし一つでも当てはまらないなら、読み進める価値があるかもしれません」と付け加える

### 第3部: 恐怖パート（行動しないリスク）｜ 3行

**見出し**: `#### ○○は危険です` または `#### 実は危ない○○`（記事内容に合わせて変更）

**活用要素**: 恐怖型訴求（関係レベル低に有効）

**構成**:

- ☠ 怖いことが起こる土壌ができる（リスクの兆候）
- ☠ やがて、こんなきっかけが発生しうる（具体的なトリガー）
- ☠ ついにこんな怖いことが起こる（最悪のシナリオ）

**注意**: 最終的には技術的負債やユーザー離れになるが、それを直接言及せず、それにつながりうるものを上げる

### 第4部: 希望パート（解決策の提示）｜ 3-4行

**見出し**: `#### ○○という明るい未来`（記事内容に合わせて変更）

**活用要素**: 希望型訴求、希少性、社会的証明、権威

**構成**:

1. ✅ この記事を読めば、○○という明るい未来があります（希望提示）
2. ✅ 具体的には、△△という方法で××を実現する**設計図**を手に入れられます（具体性＋価値）
3. ✅ この方法は、□□のプロジェクトで実証済みです（社会的証明／権威）
4. ✅ この情報は、**巷の情報では決して得られない**実践知です（希少性）

### 第5部: Bridge（信頼の橋渡し）｜ 2-3行

**見出し**: `#### 私も同じでした` または `#### このブログもそうでした`（記事内容に合わせて変更）

**活用要素**: 権威、返報性、コミットメント、好意（自己開示）

**構成**:

1. 【権威／自己開示】筆者も過去に同じ問題で悩み、○○のプロジェクトで解決した経験があります
2. 【返報性】この記事で、基本的な考え方と明日から試せるTipsを持ち帰れるように書きました
3. 【コミットメント】深掘りしたい方は、詳細な実装方法を確認できます
