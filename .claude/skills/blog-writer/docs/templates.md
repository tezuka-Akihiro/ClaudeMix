# 記事テンプレート定義

ClaudeMixブログで使用する2つの記事テンプレートを定義します。

## テンプレート選択ガイド

| テンプレート | 使用ケース | 記事の割合 |
|:---|:---|:---|
| 標準テンプレート | 機能改修記事、新機能追加記事、まとめ記事 | 30-40% |
| トラブルシューティングテンプレート | トラブルシュート記事 | 60-70% |

---

## テンプレート1: 標準テンプレート

### 使用ケース

- 機能改修記事
- 新機能追加記事
- まとめ記事（月次振り返り等）

### 構成

```markdown
### 開発の進捗

- **Before**: 着手前の状態を簡潔に説明
- **Current**: 現在の状態を簡潔に説明
- **Next**: 次に行うことを簡潔に説明

### 具体的なタスク

- **Before**: 着手前のタスク内容を3行以内で表現
- **Current**: 現在取り組んでいるタスク内容を3行以内で表現
- **Next**: 次に取り組むタスク内容を3行以内で表現

### 課題と解決策

（課題の概要をここに記述します）

#### 工夫したこと

（課題解決のために試したことや、設計上の工夫などを具体的に記述します）

#### ぶつかった壁

（発生したエラーや、AIとの認識ズレ、手戻りなど、問題となった点を具体的に記述します）

#### 解決方法

（最終的にどのように解決したのかを簡潔に記述します）

### コード抜粋

（解決策を示す、再現性のあるコードスニペットを挿入します）

```typescript
// 例: 実際に動作するコード
export function exampleFunction() {
  // コメントで説明を追加
  return "result";
}
```

### 今回の学びと感想

（今回の経験から得られた学びを記述します）

同じような課題で悩んだ方はいませんか？
もっと良い解決方法があれば教えてください！
```

### ポイント

- **Before/Current/Next**: 開発の流れを時系列で示す
- **工夫/壁/解決**: 試行錯誤のプロセスを正直に記述
- **コード抜粋**: 実際に動作する、再現性のあるコードを掲載
- **学び**: 技術的な学びを言語化する

---

## テンプレート2: トラブルシューティングテンプレート

### 使用ケース

- トラブルシュート記事
- バグ修正記事
- エラー解決記事

### 構成

```markdown
### 📝 概要

（問題の全体像を2-3文で記述します）

#### 発生環境

- **フレームワーク**: Remix v2
- **ホスティング**: Cloudflare Pages/Workers
- **ビルドツール**: Vite
- **その他**: （必要に応じて追加）

---

### ⚠️ 問題の発見と症状

（いつ、何をしたら問題が起きたのかを記述します）

**エラーメッセージ:**

```bash
ここにエラーメッセージを貼り付け
```

**症状:**

- （具体的な症状を箇条書き）
- （画面の表示、コンソールの出力など）

---

### 🔍 調査と試行錯誤のプロセス

#### 仮説1: ○○が原因ではないか？

**試したこと:**

（検証のために実施した内容）

**結果:**

（成功 / 失敗）

**分かったこと:**

（この検証から分かったこと）

#### 仮説2: △△の設定ミスを試す

**試したこと:**

（検証のために実施した内容）

**結果:**

（成功 / 失敗）

**分かったこと:**

（この検証から分かったこと）

---

### 💡 根本原因の特定

（最終的に特定した根本原因を1-2文で明確に記述します）

**原因:**

（技術的な説明）

**なぜこの問題が起きたか:**

（背景や設計上の理由）

---

### 🔧 解決策

（具体的なコード変更や設定変更を記述します。diff形式推奨）

```diff
# ファイル: path/to/file.ts

- // 修正前のコード
+ // 修正後のコード
```

**変更のポイント:**

- （変更の意図を説明）
- （なぜこの変更で解決するのか）

---

### 🎓 学んだこと・まとめ

#### 技術的な学び

- （この問題から学んだ技術的なポイント）
- （他の開発者にも役立つ知見）

#### 今後のベストプラクティス

- （再発防止策）
- （同様の問題を避けるための指針）

---

### 🔗 関連リソース

- （参考にした公式ドキュメント）
- （関連する記事やIssue）
- （GitHub PR等）

---

同じような問題で困っている方の参考になれば幸いです！
```

### ポイント

- **絵文字の使用**: セクションの視認性を高める（📝⚠️🔍💡🔧🎓🔗）
- **試行錯誤の記録**: 失敗した仮説も含めて記述（他の読者の参考になる）
- **diff形式**: コード変更を明確に示す
- **根本原因の明確化**: 「何が」「なぜ」問題を起こしたのかを説明
- **再発防止策**: 今後のベストプラクティスを提示

---

## 共通ルール（両テンプレート）

### 文体

- ですます調で統一
- 文のまとまりで適度に改行を入れる
- 専門用語は初出時に説明を加える

### フォーマット

- 横線（`---`）はフロントマターとの区切り以外では使用しない
- コードブロックには必ず言語指定（例: ` ```typescript `）
- 見出しは `###` (h2) から `#####` (h4) を使用
- `###` (h2) の見出しは記事全体で10個程度に収める

### コード

- **再現性**: 実際に動作する、再現性のあるコードを掲載
- **コメント**: 重要な部分にはコメントで説明を追加
- **diff形式**: 変更箇所が明確になるよう推奨

### リンク

- リンクは必要最小限に留める
- 記事の主題を補足する参考リンクのみ掲載
- 外部リンクには説明を追加（何のリンクか分かるように）

### 最後の一文

記事の最後には、読者への問いかけを含めてください：

**標準テンプレート:**
```markdown
同じような課題で悩んだ方はいませんか？
もっと良い解決方法があれば教えてください！
```

**トラブルシューティングテンプレート:**
```markdown
同じような問題で困っている方の参考になれば幸いです！
```
