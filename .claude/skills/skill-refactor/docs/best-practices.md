## スキル設計

### 3層構成の原則

```text
.claude/skills/{skill-name}/
├── SKILL.md             # 手順書（Workflow）
├── prompt-templates/    # プロンプト（Template）
└── docs/                # 参照ドキュメント（Knowledge）
```

| 層 | 役割 | 内容 |
| :--- | :--- | :--- |
| SKILL.md | 手順書（司令塔） | どの順番で作業し、どのプロンプトとドキュメントを呼び出すかの全体図 |
| prompt-templates/ | 金型 | コード生成や分析の際の「具体的な指示」。一貫した品質を保つための金型 |
| docs/ | 辞書 | 設計基準、Web標準の規約、過去にハマった「失敗パターン」の蓄積 |

## プロンプト品質

### 参照ガイド

- `content/blog/posts/prompts-guide.md` - Claude公式プロンプトガイドの要約

### 6技術の階層

1. **明確性と直接性** ← 最も広く効果的な基盤
2. **マルチショット例示** ← 複数の例を通じた学習
3. **思考の連鎖（CoT）** ← "Let Claude think"による推論促進
4. **XMLタグ活用** ← 構造化された入力の提供
5. **システムプロンプト** ← Claudeにロールを割り当て
6. **レスポンス事前入力** ← 出力形式の誘導

### 各技術の適用例

#### 明確性と直接性

```text
悪い例: このコードを改善して
良い例: このTypeScript関数を以下の点で改善してください:
       1. 型安全性の向上（anyを排除）
       2. エラーハンドリングの追加
```

#### マルチショット例示

```text
良い例:
| ターゲット属性 | 32歳男性、SaaS企業のインサイドセールス |

悪い例:
| ターゲット属性 | 30代男性、営業職 | ← 抽象的すぎる
```

#### 思考の連鎖（CoT）

```text
以下の順序で段階的に考えてください：

Step 1: ターゲットの「最も深刻な悩み」を特定する
Step 2: その悩みが解決された「理想の状態」を描写する
Step 3: 1と2のギャップを埋める「心理的ハードル」を洗い出す
```

#### XMLタグ活用

```xml
<image_generation_request>
  <role>あなたはプロの漫画家のアシスタントです</role>
  <style>
    <visual>日本のビジネス漫画風</visual>
  </style>
</image_generation_request>
```

#### システムプロンプト

```text
## AI役割定義

あなたはビジネスアナリスト兼UXリサーチャーです。
```

## AI共進化のサイクル

スキルは一度作って終わりではなく、以下の流れで育てる：

1. **実行**: 定義した手順でAIに作業させる
2. **学び**: 発生したエラーや、より良い解決策を docs/ に追記
3. **改善**: prompt-templates/ の精度を上げる
4. **資産化**: skills/ フォルダ自体をナレッジベースとする
