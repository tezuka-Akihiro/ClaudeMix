
# 📄 機能ドキュメント：開発ロードマップ（Development Roadmap）

## 1. 概要と目的

この機能は、開発中のWebサービスにおける開発項目、進捗、および将来的な計画を時系列で一元管理し、視覚化することを目的とします。

## 2. 機能要件

| No. | 機能名 | 内容と目標 | 実装技術 |
| :-- | :--- | :--- | :--- |
| 1 | メイン表示 | ガントチャート | 開発項目をタスクとして定義し、日単位の期間を持つ時間軸で表示する。 | Mermaid (`gantt`) |
| 2 | 過去/未来の区別 | 進捗ステータス | 完了したログ（過去）と、未着手・進行中の計画（未来）を色とセクションで明確に分離する。 | Mermaid (`done`, `active`), カスタムCSS |
| 3 | 詳細との連携 | クリックリンク | ガントチャート上の各開発項目（タスクバー）をクリックすると、対応する詳細な開発ログ記事へ遷移する。 | Mermaid (`click`), HTML (`<a>`) |
| 4 | アクセス導線 | ヘッダーメニュー | サイトのどのページからでもアクセスできるよう、ヘッダーナビゲーションに専用リンクを設置する。 | HTML/Remix ルーティング |

## 3. 実装の設計方針（3ステップ MVP）

本機能は、迅速な立ち上げと「コーディング上での品質保証」を重視し、以下の3ステップでMVPを完成させます。

### ステップ1：ロードマップ記事テンプレート作成

> **目標**: Mermaidをブラウザで正しくレンダリングできるページ構造を確立する。

*   **ファイル**: `app/routes/roadmap.tsx` など
*   **実装内容**:
    *   Remixのルートコンポーネント内に、Mermaidライブラリ（例：CDNまたはnpmパッケージ）を組み込む。
    *   ガントチャートのMermaid記法を記載するエリアを用意し、レンダリング処理を実行する。
    *   ページ内に、ロードマップの目的と**凡例（色分けの意味など）**を記載するセクションを追加する。

### ステップ2：ガントチャートレンダリングスタイル（カスタムCSS）

> **目標**: 過去と未来の識別性を高め、視覚的な地図としての品質を向上させる。

*   **実装内容**:
    *   `done` (過去) のタスクバーの色を落ち着いた色（例：薄いグレー）に設定する。
    *   `active` (進行中) のタスクバーの色を強調色（例：緑や青）に設定する。
    *   タスク名にリンクが設定されている場合に、マウスオーバーでクリック可能であることを示すスタイル（カーソル変更、下線など）を追加する。
    *   （優作さんの既存の設計に合わせ、可能な場合は）「過去のログ」と「今後の計画」を囲むセクションをCSSで区別する。

### ステップ3：ヘッダーメニュー追加

> **目標**: ユーザーが開発ロードマップに簡単にアクセスできるようにする。

*   **実装内容**:
    *   サイト共通のヘッダーナビゲーションコンポーネントに、「開発ロードマップ」という名称でリンクを追加する。
    *   リンク先をステップ1で作成したテンプレートのルート（例：`/roadmap`）に設定する。

## 4. 開発ログ記事との連携（データフロー）

| フロー | 詳細 |
| :--- | :--- |
| **入力** | 開発項目（タスク名）、開始日、期間、ステータス、対応する開発ログ記事のURL（すべてMermaid記法で定義）。 |
| **処理** | Mermaidが記法を解析し、SVG形式のガントチャートを生成。各タスクに`click`属性に基づき、詳細記事へのハイパーリンク（`<a>`タグ）を埋め込む。 |
| **出力** | 視覚化された「開発ロードマップ」と、各項目から詳細な開発ログ記事への直接的なナビゲーション。 |
