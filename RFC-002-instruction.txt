📌 タスク概要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

app/specs/shared/ 配下に、サービス横断で使用する共通specファイル群を新規作成し、
spec-loaderを拡張してこれらを読み込めるようにする。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 目的
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. SSoT原則の完全実装: サービス横断値を1箇所に集約
2. 保守性向上: 共通設定変更時の修正箇所を削減（7箇所→1箇所）
3. 拡張性確保: 今後追加されるサービスでも即座に共通設定を利用可能

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 1: 準備】

1. RFC-002ドキュメントを読んで全体像を把握
2. 既存のspec構造を確認（blog/*, account/*）
3. 新規ブランチ作成: git checkout -b claude/rfc-002-shared-spec-XXXXX

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 2: sharedディレクトリとspecファイル作成】

1. ディレクトリ作成: mkdir -p app/specs/shared

2. validation-spec.yaml を作成（RFC-002の82-113行参照）
   - email, password, username, url のバリデーションルール定義
   - 既存specファイルからの移行元を確認

3. responsive-spec.yaml を作成（RFC-002の127-164行参照）
   - breakpoints, grid_columns, spacing, font_sizes, container定義

4. server-spec.yaml を作成（RFC-002の177-210行参照）
   - loader, action, rate_limit, cache, security設定

5. types.ts を作成（RFC-002の227-373行参照）
   - ValidationSpec, ResponsiveSpec, ServerSpec, ProjectSpec型定義

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 3: spec-loader拡張】

1. app/spec-loader/specLoader.server.ts を編集
   - loadSharedSpec<T>(specName: string): T 関数を追加
   - getSharedSpec を generated/specs.ts からインポート

2. コード例（RFC-002の382-407行参照）:

import { getSpec, getSharedSpec } from '~/generated/specs';

export function loadSpec<T>(featurePath: string): T {
  return getSpec<T>(featurePath);
}

export function loadSharedSpec<T>(specName: string): T {
  return getSharedSpec<T>(specName);
}

【Phase 4: generate-specs.js拡張】

1. scripts/prebuild/generate-specs.js を編集

2. sharedSpecs オブジェクトを追加（RFC-002の447-485行参照）:
   - sharedディレクトリの処理を追加
   - project-spec.yaml → 'project'
   - validation-spec.yaml → 'validation'
   - responsive-spec.yaml → 'responsive'
   - server-spec.yaml → 'server'

3. TypeScript生成部分を更新（RFC-002の509-560行参照）:
   - sharedSpecs 定義を追加
   - getSharedSpec<T>(specName: string): T 関数を追加
   - getAllSharedSpecNames(): string[] 関数を追加
