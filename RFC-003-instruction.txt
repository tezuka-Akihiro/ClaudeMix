【Phase 1: 準備】

1. RFC-003ドキュメントを読んで全体像を把握
2. RFC-002で作成したshared specsを確認
3. 新規ブランチ作成: git checkout -b claude/rfc-003-migrate-shared-XXXXX

【Phase 2: data-io層の拡張（新規関数追加）】

重要: 既存関数は変更せず、新規関数を追加して破壊的変更を回避

1. blog/posts 関連（RFC-003の144-185行参照）:

   app/data-io/blog/posts/loadPostsSpec.server.ts に追加:

   export function loadPostsServerConfig() {
     const serverSpec = loadSharedSpec<ServerSpec>('server');
     return {
       timeout: serverSpec.loader.timeout
     };
   }

   export function loadPostsResponsiveConfig() {
     const responsiveSpec = loadSharedSpec<ResponsiveSpec>('responsive');
     return {
       breakpoints: responsiveSpec.breakpoints
     };
   }

   export function loadProjectInfo() {
     const projectSpec = loadSharedSpec<ProjectSpec>('project');
     return {
       name: projectSpec.project.name,
       copyrightName: projectSpec.project.name
     };
   }

2. blog/common 関連（RFC-003の210-252行参照）:

   app/data-io/blog/common/loadBlogConfig.server.ts を更新:
   - loadSharedSpec('project') を使用
   - copyright表示にプロジェクト名を使用

3. account/authentication 関連（RFC-003の254-303行参照）:

   app/data-io/account/authentication/validateCredentials.server.ts を更新:
   - validateEmail(): shared/validation-spec から読み込み
   - validatePassword(): shared/validation-spec から読み込み
   - getPasswordRequirements(): 新規関数追加

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 3: routes層の段階的移行】

2. 動作確認を各ファイルごとに実施

【Phase 4: 既存specファイルからの削除】

routes層の移行完了後、specファイルから重複を削除:

1. project名・copyright名の削除（RFC-003の40-57行参照）:
   - blog/posts-spec.yaml: project セクション削除
   - blog/common-spec.yaml: copyright_name 削除

2. timeout設定の削除（RFC-003の60-78行参照）:
   - 全7ファイル（blog/*, account/*）から server_io.loader.timeout 削除

3. responsive設定の削除（RFC-003の80-103行参照）:
   - blog/common-spec.yaml: breakpoints 削除
   - blog/posts-spec.yaml: breakpoints 削除（grid_columnsは保持）
   - account/common-spec.yaml: breakpoints 削除

4. validation設定の削除（RFC-003の106-137行参照）:
   - account/authentication-spec.yaml: validation.email, password 削除
   - account/profile-spec.yaml: new_email, new_password 削除
   - current_password は保持（profile固有）
