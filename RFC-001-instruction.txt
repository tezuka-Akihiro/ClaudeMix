📌 タスク概要

scripts/project.toml を YAML 形式に変換し、app/specs/shared/project-spec.yaml
として再配置することで、プロジェクト全体の設定フォーマットを YAML に統一します。

🎯 目的
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. セキュリティ向上: TOMLパーサー（@iarna/toml）依存を排除
2. 保守性向上: YAML統一によるフォーマット一貫性確保
3. 拡張性確保: 将来のサービス横断spec導入の基盤整備

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 1: 準備（必須）】

1. RFC-001ドキュメントを読んで全体像を把握
2. 現在のscripts/project.tomlの内容を確認
3. 新規ブランチ作成: git checkout -b claude/rfc-001-yaml-migration-XXXXX

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 2: YAMLファイル作成】

1. app/specs/shared/ ディレクトリ作成
2. scripts/project.toml を YAML 形式に変換
   - RFC-001の「フォーマット変換例」（78-163行目）を参考
   - メタデータセクション追加（version, migrated_from, migration_date）
3. app/specs/shared/project-spec.yaml として保存
4. app/specs/shared/project-spec.example.yaml も作成

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 3: スクリプト修正】

1. scripts/generate/scaffold-and-lint.js を修正
   - import文変更: @iarna/toml → js-yaml
   - ファイルパス変更
   - loadProjectConfig() 関数を YAML 対応に変更
     （RFC-001の202-223行目参照）
   - YAML構造をTOML互換形式に変換する処理を追加

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 4: テスト更新】

1. scripts/generate/scaffold-and-lint.test.js のテストケース確認
2. 必要に応じてテストを更新（YAML読み込みに対応）
3. テスト実行: npm test scripts/generate/scaffold-and-lint.test.js

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 5: 統合テスト】

1. コード生成機能のテスト:
   npm run generate -- --type ui --ui-type component --service blog --section posts --name TestComponent

2. 生成されたファイルが正しいか確認
3. 型チェック: npm run typecheck
4. 全テスト実行: npm test

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 6: パッケージ削除】

1. TOMLパッケージ削除: npm uninstall @iarna/toml
2. package.json から依存関係が削除されたことを確認
3. npm install でロックファイル更新

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Phase 7: クリーンアップ】

1. 旧ファイル削除:
   - scripts/project.toml
   - scripts/project.example.toml
2. 最終テスト実行（全テストパス確認）
